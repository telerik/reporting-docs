# Options of the Angular Report Viewer

| Property | Description |
| ------ | ------ |
| [authenticationToken]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/authenticationtoken%})|If provided, a Bearer token will be set in the Authorization header for every request to the REST service.|
| [checkedButtonClass]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/checkedbuttonclass%})|CSS class added to buttons when a command is in a checked state.<br>Used with data attributes for external command buttons.|
| [containerStyle]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/containerstyle%})|CSS styles object for the report viewer's container element.|
| [disabledButtonClass]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/disabledbuttonclass%})|CSS class added to buttons when a command is in a disabled state.<br>Used with data attributes for external command buttons.|
| [documentMapAreaPosition]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/documentmapareaposition%})|Specifies where the Document Map should be displayed.<br>Available values: "RIGHT", "LEFT".|
| [documentMapVisible]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/documentmapvisible%})|Determines whether the viewer's document map is displayed when bookmarks are defined.|
| [enableAccessibility]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/enableaccessibility%})|Determines whether the viewer should provide accessibility features support.|
| [id]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/id%})|Sets the unique identifier of the ReportViewer instance.<br>If not specified, the id of the target HTML element will be used.<br>The id is used to identify the client session when persistSession is enabled.|
| [initialPageAreaImageUrl]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/initialpageareaimageurl%})|The image URL for the PageArea background image.<br>Used only when parameter values are missing or invalid.<br>The image should be in PNG, GIF, or JPG file format.|
| [keepClientAlive]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/keepclientalive%})|Determines whether the client will be kept alive.<br>When true, prevents client expiration by continually sending requests to the server.|
| [pageMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/pagemode%})|Sets if the report is displayed in single page or continuous scroll mode.<br>- "SINGLE_PAGE" - Only one page loaded in viewport. Pages are changed using the page navigation buttons in the viewer's toolbar.<br>- "CONTINUOUS_SCROLL" - Multiple pages can be loaded in viewport. Pages can be changed using the scroll wheel, by dragging the scroll bar, or by clicking the page navigation buttons in the viewer's toolbar.|
| [parameterEditors]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/parametereditors%})|Allows defining custom editors for report parameters.<br>Array of parameter editor configurations.<br>Each object in the array represents a parameter editor factory for creating editors suitable to edit a specific report parameter configuration.<br>Each factory object contains two methods: <br>`match`<br> and <br>`createEditor`<br>.<br><br>The <br>`match`<br> method accepts a report parameter to be edited as an argument and returns a boolean value which indicates whether the parameter editor is suitable for this parameter.<br>The parameter argument exposes the following properties of the report parameter:<br>- availableValues (array of name-value pairs): represents a list of predefined parameter values and their labels.<br>- multivalue	(boolean): specifies whether the parameter can hold more than one value.<br>- allowBlank (boolean): specifies whether the parameter allows an empty string to be passed (only for parameters of type string).<br>- allowNull (boolean): specifies whether the parameter allows a null value to be passed from its editor<br>- autoRefresh (boolean): specifies whether the report viewer should auto-update the report when its value has changed.<br>- hasChildParameters (boolean): specifies whether the parameter is a cascading one (has dependencies).<br>- id (string): represents the parameter’s unique identifier.<br>- isVisible (boolean): determines whether the parameter editor is visible or not.<br>- label (string): specifies the label for the parameter editor.<br>- name (string): specifies the name of the parameter editor.<br>- text (string): specifies the text for the parameter editor.<br>- type (string): specifies the type of the parameter editor. Possible values: "INTEGER", "FLOAT", "STRING", "DATETIME", "BOOLEAN".<br>- value (object): specifies the value of the parameter.<br><br>The main work for creating and utilizing the parameter editor is done in the <br>`createEditor`<br> method. Its purpose is to create the parameter editor UI and wire it to the <br>`parameterChanged`<br> callback when a new value is selected. The return result is a new object containing the <br>`beginEdit`<br> method which is the entry point for creating the editor from the viewer.<br>The 'createEditor' method accepts two arguments - <br>`placeholder`<br> and <br>`options`<br>. The 'placeholder' argument is the HTML element which the parameter editor UI is appended to, and the 'options' arrgument is an object containing the <br>`parameterChanged(parameter, value)`<br> callback function which should be called when the parameter value is changed.<br>The 'createEditor' method should return a parameter editor object with the following properties:<br>- 'beginEdit(parameter)' - a method which accepts a report parameter object and initializes the editor with the parameter's settings. The parameter object has the same properties as described above.<br>- 'addAccessibility(parameter)' - an optional method which accepts a report parameter object and adds accessibility features to the editor and populates its string resources.<br>- 'setAccessibilityErrorState(parameter)' - an optional method which accepts a report parameter object and sets the error state of the parameter editor's accessibility functionality and its error attributes.<br>- 'enable(enable)' - an optional method which accepts a boolean value and enables or disables the parameter editor.<br>- 'clearPendingChange' - a method that would be invoked when the parameter changes.<br>- 'destroy' - a method invoked to destroy the parameter editor.|
| [parameters]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/parameters%})|Allows defining parameter options for report parameters.<br>Available properties:<br>- editors (object, optional) - Defines the editor type for single and multi-select parameters.<br>The 'editors' object has two properties for the two parameter types:<br> - singleSelect (string, optional) - Editor type for single-select parameters. Possible values: "COMBO_BOX" (Kendo UI ComboBox) or "LIST_VIEW" (Kendo UI ListView). Default: "LIST_VIEW".<br> - multiSelect (string, optional) - Editor type for multi-select parameters. Possible values: "COMBO_BOX" (Kendo UI ComboBox) or "LIST_VIEW" (Kendo UI ListView). Default: "LIST_VIEW".|
| [parametersAreaPosition]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/parametersareaposition%})|Specifies where the Parameters Area should be displayed.<br>Available values: "RIGHT", "TOP", "LEFT", "BOTTOM".|
| [parametersAreaVisible]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/parametersareavisible%})|Determines whether the viewer's parameters area is displayed by default when parameter editors exist.|
| [persistSession]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/persistsession%})|Sets whether the viewer's client session should be persisted between page refreshes.<br>Session is stored in browser's sessionStorage and lasts for the page session duration.|
| [printMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/printmode%})|Specifies how the viewer should print reports.<br>- "AUTO_SELECT" - automatically decide based on browser and PDF plugin availability.<br>- "FORCE_PDF_FILE" - download document as PDF file with print script.<br>- "FORCE_PDF_PLUGIN" - always use PDF plugin.|
| [reportServer]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/reportserver%})|Sets the configuration details for connecting to a Telerik Report Server.<br><br>The authentication method depends on the Report Server version:<br>- Report Server for .NET: Supports Token authentication (getPersonalAccessToken) or username/password authentication. The token can be from any user, including the Guest user.<br>- Report Server for .NET Framework 4.6.2: Supports username/password authentication or Guest account (url only, if Guest is enabled on the server).<br><br>Has the following properties:<br>- url (string, required) - The URL to the Telerik Report Server instance.<br>- getPersonalAccessToken (function, optional) - A callback function that returns the Token for authentication against the Telerik Report Server for .NET instance wrapped in a Promise. This is the recommended authentication method for Report Server for .NET. The token can be from any user account, including the Guest user. Only supported by Report Server for .NET.<br>- username (string, optional) - A registered username in the Report Server that will be used to get access to the reports. Supported by both Report Server for .NET and Report Server for .NET Framework 4.6.2. If omitted when connecting to Report Server for .NET Framework 4.6.2 with Guest enabled, the built-in Guest account will be used.<br>- password (string, optional) - The password for the provided username. Supported by both Report Server for .NET and Report Server for .NET Framework 4.6.2. Can be omitted only when connecting with the Guest account to Report Server for .NET Framework 4.6.2.<br><br>Required when not using serviceUrl.|
| [reportSource]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/reportsource%})|Sets the report and initial report parameter values to be displayed.<br>Available properties:<br>- report (string) - A string that represents a report on the back end.<br>- parameters (JSON object) - An object with name/value properties equal to the report parameters’ names and values used in the report definition.|
| [scale]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/scale%})|Sets the scale factor for the report pages.<br>Takes effect when scaleMode is set to "SPECIFIC".|
| [scaleMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/scalemode%})|Sets how the report pages should be scaled.<br>- "FIT_PAGE_WIDTH" - pages scaled to fit entire width in viewport.<br>- "FIT_PAGE" - pages scaled to fit entire page in viewport.<br>- "SPECIFIC" - pages scaled with the scale value.|
| [searchMetadataOnDemand]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/searchmetadataondemand%})|Determines whether search metadata will be delivered on demand (true) or by default (false).|
| [selector]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/selector%})|A selector used in conjunction with the data attributes.<br>Required when commands are attached to elements outside the viewer via data attributes.|
| [sendEmail]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/sendemail%})|Configuration for the Send Email functionality.<br>Available properties:<br>- enabled (bool, required) - Indicates whether to show the Send Mail Message toolbar button. Default value: false;<br>- from (string, optional) - E-mail address used for the MailMessage FROM value;<br>- to (string, optional) - E-mail address used for the MailMessage TO value;<br>- cc (string, optional) - E-mail address used for the MailMessage Carbon Copy value;<br>- subject (string, optional) - A string used for the MailMessage Subject value;<br>- body (string, optional) - Sentences used for the MailMessage Content value;<br>- format (string, optional) - The preselected report document format.|
| [serviceUrl]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/serviceurl%})|Sets the address of the Report REST Service.<br>Required when not using reportServer configuration.|
| [templateUrl]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/templateurl%})|Sets the address of the HTML page that contains the viewer templates.<br>If omitted, the default template will be downloaded from the REST service.|
| [viewMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/angular-report-viewer/api-reference/reportviewer/options/viewmode%})|Sets if the report is displayed in interactive mode or print preview.<br>- "INTERACTIVE" - enables drill-down interactivity.<br>- "PRINT_PREVIEW" - report paged according to page settings.|

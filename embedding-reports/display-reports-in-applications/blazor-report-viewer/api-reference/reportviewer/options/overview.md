# Options of the Blazor Report Viewer

| Property | Description |
| ------ | ------ |
| [AccessibilityKeyMap]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/accessibilitykeymap%})|Gets or sets the key shortcuts map, used when the report viewer is in accessible mode.|
| [AuthenticationToken]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/authenticationtoken%})|Gets or sets the encoded authentication token used to authenticate the requests.|
| [CheckedButtonClass]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/checkedbuttonclass%})|Gets or sets the class to be used by buttons in the checked state.|
| [ClientEvents]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/clientevents%})|Gets or sets the viewer's event handler function names as strings.<br>The function name should be fully qualified, e.g. "window.trvEventHandlers.exportBegin",<br>or just "trvEventHandlers.exportBegin" ("window" qualifier is optional).<br>The event handler functions should be implemented inside wwwroot/index.html (Blazor WebAssembly) or Pages/_Host.cshtml (Blazor Server).<br>The JavaScript API of the event handlers is available in <br>https://docs.telerik.com/reporting/html5-report-viewer-reportviewer-events-ready.|
| [Commands]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/commands%})|Gets the available report viewer commands.|
| [DisabledButtonClass]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/disabledbuttonclass%})|Gets or sets the class to be used by buttons in the disabled state.|
| [DocumentMapAreaPosition]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/documentmapareaposition%})|Gets or sets the document map area position.<br>Left (Default).<br>Right.|
| [DocumentMapVisible]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/documentmapvisible%})|Gets or sets  a value indicating whether the viewer's document map is initially visible.|
| [EnableAccessibility]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/enableaccessibility%})|Gets or sets a value indicating whether the accessibility features of the report viewer and its contents are enabled.|
| [InitialPageAreaImageUrl]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/initialpageareaimageurl%})|Gets or sets the image URL for the PageArea background image. Used only when the parameter values are missing or invalid.|
| [KeepClientAlive]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/keepclientalive%})|Gets or sets a value indicating whether the client will be kept alive.<br><br> When set to true a request will be sent to the server to stop the client from expiring, determined by the ClientSessionTimeout server configuration<br><br> When set to false, the client will be left to be expired|
| [PageMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/pagemode%})|Gets or sets the page mode.<br>SinglePage - Displays a single report page.<br>ContinuousScroll - Displays multiple report pages. Pages are loaded on-demand while scrolling.|
| [ParameterEditors]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/parametereditors%})|Gets or sets the user-defined custom editors for the report parameters. Each custom editor requires<br>a function name for its `Telerik.ReportViewer.Blazor.ParameterEditor.Match` and `Telerik.ReportViewer.Blazor.ParameterEditor.CreateEditor` properties.<br>The function names should be fully qualified, e.g. "window.trvDateTimeCustomEditor.match",<br>or just "trvDateTimeCustomEditor.match" ("window" qualifier is optional).<br>The functions should be implemented inside the wwwroot/index.html (Blazor WebAssembly) or Pages/_Host.cshtml (Blazor Server).|
| [Parameters]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/parameters%})|Gets or sets the options of the built-in parameter editors.|
| [ParametersAreaPosition]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/parametersareaposition%})|Gets or sets the parameters area position.<br>Right (Default).<br>Left.<br>Top.<br>Bottom.|
| [ParametersAreaVisible]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/parametersareavisible%})|Gets or sets a value indicating whether the viewer's parameters area is initially visible.|
| [PersistSession]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/persistsession%})|Gets or sets a value indicating whether the viewer's client state will be persisted between the page refreshes/postbacks.<br>The state is stored in the browser's sessionStorage and is available for the duration of the page session.|
| [PrintMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/printmode%})|Gets or sets the print mode of the viewer.<br>Three modes exist currently:<br>AutoSelect - The viewer automatically determinse how to print: via the browser's PDF plug-in or exporting to a PDF file.<br>ForcePDFPlugin - The viewer always uses the PDF plugin regardless of the browser's version and settings.<br>ForcePDFFile - The viewer always exports the report document to PDF format with 'automatic print on open' script enabled.|
| [ReportServer]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/reportserver%})|Gets or sets the connection data of the report server that hosts the reports.<br>When a Report Server connection is used, the ServiceUrl property should not be set.<br>For more information see http://www.telerik.com/help/reporting/html5-report-viewer-howto-use-it-with-reportserver.html.|
| [ReportSource]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/reportsource%})|Gets or sets the [`Telerik.ReportViewer.Blazor.ReportSourceOptions`]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/types/reportsourceoptions%})<br>for the report displayed in the viewer.|
| [Scale]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/scale%})|Gets or sets zoom ratio of the report.<br>ScaleMode needs to be set to Specific for this property to take effect.<br>1.0 is equal to 100%, i.e. the original size of the report.|
| [ScaleMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/scalemode%})|Gets or sets the scale mode of the viewer.<br>Three modes exist currently:<br>FitPage - The whole report will fit on the page (will zoom in or out), regardless of its width and height.<br>FitPageWidth - The report will be zoomed in or out so that the width of the screen and the width of the report match.<br>Specific - Uses the scale to zoom the report in and out.|
| [SearchMetadataOnDemand]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/searchmetadataondemand%})|Gets or sets a value indicating whether the search metadata will be delivered on demand or by default.|
| [Selector]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/selector%})|Gets or sets a selector that is used in conjunction with data attributes (https://docs.telerik.com/reporting/html5-report-viewer-data-attributes).<br>Whenever a command is attached to an element outside of the report viewer via a data-attribute this selector must be provided.|
| [SendEmail]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/sendemail%})|Gets or sets the default send e-mail message options.|
| [ServiceUrl]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/serviceurl%})|Gets or sets the URL of the service which will provide the report viewer with reports<br>(relative or absolute).<br>The service must be properly configured so that the report viewer can<br>successfully communicate with the server.<br>For more information on how to configure the service please check<br>http://www.telerik.com/help/reporting/telerik-reporting-rest-conception.html.|
| [TemplateUrl]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/templateurl%})|Gets or sets the url for the report viewer template. A customized template can be provided as described in<br>http://www.telerik.com/help/reporting/html5-report-viewer-templates.html.|
| [ViewerId]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/viewerid%})|Gets or sets the unique identifier of the ReportViewer instance.|
| [ViewMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/blazor-report-viewer/api-reference/reportviewer/options/viewmode%})|Gets or sets the view mode.<br>PrintPreview - Displays the paginated report as if it is printed on paper. Interactivity is disabled.<br>Interactive - Displays the report in its original width and height with no paging. Interactivity is enabled.|

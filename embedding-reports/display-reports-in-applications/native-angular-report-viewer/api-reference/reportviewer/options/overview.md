# Options of the Native Angular Report Viewer

| Property | Description |
| ------ | ------ |
| [authenticationToken]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/authenticationtoken%})|A bearer token to be added in the Authorization header of each request.<br>Used for authentication with the reporting service.|
| [contentTabIndex]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/contenttabindex%})|Sets the tab index for the report viewer content.|
| [documentMapVisible]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/documentmapvisible%})|Controls the visibility of the document map panel if a document map is available.|
| [enableSendEmail]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/enablesendemail%})|Enables or disables the send email functionality.|
| [height]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/height%})|Sets the height of the report viewer container.|
| [keepClientAlive]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/keepclientalive%})|Determines whether to keep the client session alive.<br>When true, prevents the session from timing out during long periods of inactivity.|
| [pageMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/pagemode%})|Sets the page display mode of the viewer.|
| [parametersAreaVisible]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/parametersareavisible%})|Controls the visibility of the parameters area if the report has parameters.|
| [persistSession]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/persistsession%})|Determines whether to persist the viewer session across browser sessions.|
| [printMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/printmode%})|Sets the print mode for the report viewer.|
| [reportServer]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/reportserver%})|Configuration for Telerik Report Server connection.<br><br>The authentication method depends on the Report Server version:<br>- Report Server for .NET: Supports Token authentication (ReportServerTokenSettings) or username/password authentication (ReportServerSettings). The token can be from any user, including the Guest user.<br>- Report Server for .NET Framework 4.6.2: Supports username/password authentication (ReportServerSettings) or Guest account (url only with ReportServerSettings, if Guest is enabled on the server).<br><br>Required when serviceType is set to 'reportServer'.<br>The ReportServerSettings or ReportServerTokenSettings types should be imported from '@progress/telerik-common-report-viewer'.<br><br>ReportServerTokenSettings properties:<br>- url (string, required) - The URL to the Telerik Report Server instance.<br>- getPersonalAccessToken (function, required) - A callback function that returns the Token for authentication against the Telerik Report Server for .NET instance wrapped in a Promise. This is the recommended authentication method for Report Server for .NET. The token can be from any user account, including the Guest user. Only supported by Report Server for .NET.<br><br>ReportServerSettings properties:<br>- url (string, required) - The URL to the Telerik Report Server instance.<br>- username (string, required) - A registered username in the Report Server that will be used to get access to the reports. Supported by both Report Server for .NET and Report Server for .NET Framework 4.6.2.<br>- password (string, required) - The password for the provided username. Supported by both Report Server for .NET and Report Server for .NET Framework 4.6.2.|
| [reportSource]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/reportsource%})|Specifies the report and initial report parameter values to be displayed.<br>The report property can be a relative path to a physical file (TRDP/TRDX/TRBP) or an assembly qualified name for type definitions.<br>The ReportSourceOptions type should be imported from '@progress/telerik-common-report-viewer'.|
| [scale]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/scale%})|Sets the scale factor for the report pages.<br>The scale takes effect when the scaleMode is set to 'specific'.|
| [scaleMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/scalemode%})|Sets how the report pages are scaled.|
| [serviceType]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/servicetype%})|Specifies the type of service that the report viewer will connect to.|
| [serviceUrl]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/serviceurl%})|Sets the address of the Reporting REST Service.<br>Required when serviceType is 'REST'.|
| [viewMode]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/viewmode%})|Sets the view mode of the report viewer.|
| [width]({%slug telerikreporting/using-reports-in-applications/display-reports-in-applications/web-application/native-angular-report-viewer/api-reference/reportviewer/options/width%})|Sets the width of the report viewer container.|
